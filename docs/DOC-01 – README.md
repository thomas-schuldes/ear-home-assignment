# EAR – Dokumentation

Dieses Verzeichnis enthält alle fachlichen, technischen und architektonischen Dokumente
für die EAR-Entwicklungsaufgabe.

## Struktur

- **DOC-02 – TICKETS.md**  
  Enthält alle User Stories und technischen Tickets, abgeleitet aus den EAR-Anforderungen.

- **DOC-03 – Architektur-Dokument.md**  
  Statische und dynamische Architekturbeschreibung des Systems.

- **DOC-04 – ADR Persistenz**  
  Architekturentscheidungen (Architecture Decision Records), beginnend mit ADR 001 zur Persistenz.

- **DOC-05 – decision-sonarqube-vs-grafana.md**  
  Entscheidungsvorlage für das Management.

## Bezug zur Aufgabenstellung
Diese Dokumente basieren auf der Aufgabenbeschreibung der EAR-Entwicklungsaufgabe  
und erfüllen die Anforderung, dass alle Artefakte versioniert im Repository abgelegt werden.
